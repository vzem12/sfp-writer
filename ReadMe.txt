Программатор SFP и SFP+ by ZemtsovVA v2.3.3.

ПО созданно с целью чтения, а также полного или частичного изменения прошивки SFP и SFP+ модулей.
Возможности ПО:
	1. Чтение и вывод в графический интерфейс значений байтов регистра 0x50 I2C EEPROM SFP или SFP+ модулей в 
	расшифрованном в соответствии с SFF-8472 виде
	2. Изменение значений байтов регистра 0x50 I2C EEPROM SFP или SFP+ модулей в графическом интерйесе в 
	рашифрованном в	соответсвии с SFF-8472 виде
	3. Сохранение считанной прошивки в бинарный файл
	4. Запись измененной в графическом интерфейсе прошивки в регистр 0x50 I2C EEPROM SFP или SFP+ модулей
	5. Загрузка бинарного файла прошивки с расшировкой значений байтов в соответствии с SFF-8472 с 
	возможностью дальнейшего сохранения в бинарный файл или загрузки в регистр 0x50 I2C EEPROM SFP или SFP+ 
	модулей
	6. Ввод пароля для возможности доступа к памяти SFP в режиме записи. Некоторые SFP модули защищены от 
	записи паролем. Для разблокировки памяти необходимо вводить в определенные ячейки памяти пароль. Данное 
	ПО позволяет ввести пароль разблокировки памяти.
	7. Подбор пароля I2C EEPROM SFP или SFP+ модулей из списка (возможно сформировать любой список и вставить
	его в виде тектса с разделителем в виде символа новой строки \n)
	8. Просмотр содержимого раздела A2h памяти SFP (0x51)
	9. Расшифровка значений текущих аварий и параметров SFP

При помощи данного ПО, руководствуясь документом SFF-8472, возможна организация совместимости сторонних SFP 
модулей с брендированным оборудованием, изменение значений серийных номеров SFP, в случае их дублирования, выбор 
режимов работы модуля, в случае неверных заводских значений, изменение параметров DDM и т.д. Более полный список 
возможностей описан в документе SFF-8472. 
P.S. Следует помнить, что прошивка не изменяет физических характеристик модуля, а лишь дает возможность 
задействовать необходимые имеющиеся физические возможности модуля.

Для работы программатора необходимо собрать программатор по схеме в файле Arduino_SFP_Prog_Scheme.vsdx 
(подойдет плата Arduino nano ATmega 328p, WEMOS LOLIN 32 или WEMOS D1 mini).
Предпочтительно использовать WEMOS D1 mini т.к. для нее имеется проект печатной платы, готовой к производству - 
файл SFP_Prog_Board_WEMOS_D1_Mini.lay6

Для платы Wemos LoLin 32 использовать файл ProgrammatorSFP_2.5.ino.lolin32_0x10000.bin, 
в ПО Flash Download Tools https://www.espressif.com.cn/en/support/download/other-tools выбрать чип ESP32
Для платы Wemos D1 mini использовать файл ProgrammatorSFP_2.5.ino.d1_mini_0x01000.bin, 
в ПО Flash Download Tools https://www.espressif.com.cn/en/support/download/other-tools выбрать чип ESP8266
Для платы Arduino Nano использовать прошивку ProgrammatorSFP_2.5.ino.with_bootloader.eightanaloginputs.hex, 
в ПО XLoader https://github.com/binaryupdates/xLoader выбрать чип Duemilanove/Nano(ATmega328)
